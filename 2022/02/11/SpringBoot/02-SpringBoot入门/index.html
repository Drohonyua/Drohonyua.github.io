02、SpringBoot2入门

1、系统要求

  • Java 8 & 兼容java14 .

  • Maven 3.3+

  • idea 2019.1.2

1.1、maven设置

<mirrors>
      <mirror>
        <id>nexus-aliyun</id>
        <mirrorOf>central</mirrorOf>
        <name>Nexus aliyun</name>
        <url>http://maven.aliyun.com/nexus/content/groups/public</url>
      </mirror>
  </mirrors>
 
<profiles>
         <profile>
              <id>jdk-1.8</id>
              <activation>
                <activeByDefault>true</activeByDefault>
                <jdk>1.8</jdk>
              </activation>
              <properties>
                <maven.compiler.source>1.8</maven.compiler.source>
                <maven.compiler.target>1.8</maven.compiler.target>
                <maven.compiler.compilerVersion>1.8</maven.compiler.compilerVersion>
              </properties>
         </profile>
  </profiles>

2、HelloWorld

需求：浏览发送/hello请求，响应 Hello，Spring Boot 2

2.1、创建maven工程

2.2、引入依赖

<!--Springboot父工程，父工程也是pom文件里面帮你配置好了大多数需要的依赖-->
<parent>
      <groupId>org.springframework.boot</groupId>
      <artifactId>spring-boot-starter-parent</artifactId>
      <version>2.3.4.RELEASE</version>
</parent>
<!--web的场景启动器-->
<dependencies>
     <dependency>
         <groupId>org.springframework.boot</groupId>
         <artifactId>spring-boot-starter-web</artifactId>
     </dependency>
</dependencies>

2.3、创建主程序

/**
 * 用了这个注解的类也称为----->主程序类(所有启动的入口)
 * @SpringBootApplication：标志这是一个SpringBoot的应用
 */
@SpringBootApplication
public class MainApplication {

    public static void main(String[] args) {
        //固定写法
        SpringApplication.run(MainApplication.class,args);
    }
}

2.4、编写业务

@RestController
public class HelloController {

    //需和主程序同一文件夹下，不然会找不到路径  原因：未知
    @RequestMapping("/hello")
    public String handle01(){
        return "Hello, Spring Boot 2!";
    }


}

2.5、测试

直接运行main方法

2.6、简化配置

在resources文件夹下创建application.properties(固定这一个名字！！！！！)

server.port=8888

2.7、简化部署

<build>
     <plugins>
         <plugin>
             <groupId>org.springframework.boot</groupId>
             <artifactId>spring-boot-maven-plugin</artifactId>
         </plugin>
     </plugins>
</build>

可以把项目打成jar包，直接在目标服务器上面直接执行即可(里面自带环境)。

注意点：

  • 取消掉cmd的快速编辑模式
