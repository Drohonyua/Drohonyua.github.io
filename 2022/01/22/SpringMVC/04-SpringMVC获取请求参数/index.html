04-SpringMVC获取请求参数

SpringMVC

前端向后端传输数据

1、通过ServletAPI获取

将HttpServletRequest作为控制器方法的形参，此时HttpServletRequest类型的参数表示封装了当前请求的请求报文的对象（如果使用了占位符的方式获取参数则不能在使用这种方式获取）

页面代码:

<a th:href="@{/testServletAPI(username='admin',pwd='123456')}">test</a>

控制器代码:

@RequestMapping("/testParam")
public String testParam(HttpServletRequest request){
    String username = request.getParameter("username");
    String password = request.getParameter("password");
    System.out.println("username:"+username+",password:"+password);
    return "success";
}

2、通过控制器方法的形参获取请求参数

在控制器方法的形参位置，设置和请求参数同名的形参，当浏览器发送请求，匹配到请求映射时，在DispatcherServlet中就会将请求参数赋值给相应的形参

页面代码:

<a th:href="@{/testParams(username='long',pwd='123')}">通过控制器形参获取参数</a><br>

控制器代码:

    @RequestMapping("/testParams")
    public String testParams(String username,String pwd){
        System.out.println("username:"+username+"\n"+"pwd:"+pwd);
        return "success";
    }
    //结果：username:long
    //       pwd:123

注：

若请求所传输的请求参数中有多个同名的请求参数，此时可以在控制器方法的形参中设置字符串数组或者字符串类型的形参接收此请求参数

若使用字符串数组类型的形参，此参数的数组中包含了每一个数据

若使用字符串类型的形参，此参数的值为每个数据中间使用逗号拼接的结果

而有多个同名参数时：

页面代码：

<form th:action="@{/testParams}" method="post">
    用户名：<input type="text" name="username"><br>
    密  码：<input type="password" name="pwd"><br>
    爱  好： <input type="checkbox" name="hobby" value="a">a
            <input type="checkbox" name="hobby" value="b">b
            <input type="checkbox" name="hobby" value="c">c
    <br>
    <input type="submit" value="提交">
</form>

控制器代码:

@RequestMapping("/testParams")
//使用Stirng接收，则得到由逗号拼接的String结果
public String testParams(String username,String pwd,String hobby){
     System.out.println("username:"+username+"\n"+"pwd:"+pwd+"\n"+"hobby:"+hobby);
     return "success";
}
//结果：
//username:1
//pwd:123
//hobby:a,b
//使用String数组接收，则得到数组
@RequestMapping("/testParams")
public String testParams(String username,String pwd,String[] hobby){
     System.out.println("username:"+username+"\n"+"pwd:"+pwd+"\n"+"hobby:"+ Arrays.toString(hobby));
     return "success";
}
//结果
//username:123
//pwd:123
//hobby:[a, c]

3.@RequestParam

@RequestParam是将请求参数和控制器方法的形参创建映射关系

举例：请求中的参数的参数名为user-name，控制器中方法的形参名为username，此时前端控制器无法通过名字匹配注入值，需要使用@RequestRaram(“user-name”)来帮助注入即

@RequestMapping("/testParams")
public String testParams(@RequestParam("user_name") String username){
     System.out.println("username:"+username);
     return "success";
}

@RequestParam注解一共有三个属性：

value：指定为形参赋值的请求参数的参数名

required：设置是否必须传输此请求参数，默认值为true

若设置为true时，则当前请求必须传输value所指定的请求参数，若没有传输该请求参数，且没有设置defaultValue属性，则页面报错400：Required String parameter ‘xxx’ is not present；若设置为false，则当前请求不是必须传输value所指定的请求参数，若没有传输，则注解所标识的形参的值为null

defaultValue：不管required属性值为true或false，当value所指定的请求参数没有传输或传输的值为””时，则使用默认值为形参赋值

4、@RequestHeader

@RequestHeader是将请求头信息和控制器方法的形参创建映射关系

@RequestHeader注解一共有三个属性：value、required、defaultValue，用法同@RequestParam

    @RequestMapping("/testParams")
    public String testParams(@RequestHeader("Host")String host){
        System.out.println("host:"+host);
        return "success";
    }

5、@CookieValue

@CookieValue是将cookie数据和控制器方法的形参创建映射关系

@CookieValue注解一共有三个属性：value、required、defaultValue，用法同@RequestParam

@RequestMapping("/testParams")
public String testParams(@CookieValue("JSESSIONID")String jsessionid)
{
        System.out.println("jsesession："+jsessionid);
        return "success";
}
//结果：jsesession：0BC6264AF57749BDF09AE11BBCCCA1E3

回顾session和cookie

1.session依赖于cookie

2.session是服务端的会话技术，cookie是客户端的会话技术看，两个默认的生命周期都为从浏览器打开到浏览器关闭

3.cookie工作原理：第一次访问时由服务器创建将其放在响应头中给到浏览器，当往后再次访问时(在cookie生命周期内)浏览器会将cookie放在请求头中提交给服务器

3.工作原理：

当控制器方法调用getsession方法时(第一次调用)：

会检测请求头中是否含有JSESSIONID的cookie如果没有就会自动创建httpsession对象————–>创建一个cookie(JSESSIONID为键，随机序列为值),将httpsession对象存储到当前服务器维护的map集合中（以JSESSIONID的值即那个随机序列为键，session对象作为值）————–>再把这个cookie响应到浏览器

在cookie生命周期内当往后每次向服务器发送请求时请求头都会直接携带这个cookie

6、通过POJO获取请求参数

可以在控制器方法的形参位置设置一个实体类类型的形参，此时若浏览器传输的请求参数的参数名和实体类中的属性名一致，那么请求参数就会为此属性赋值

<form th:action="@{/testpojo}" method="post">
    用户名：<input type="text" name="username"><br>
    密码：<input type="password" name="password"><br>
    性别：<input type="radio" name="sex" value="男">男<input type="radio" name="sex" value="女">女<br>
    年龄：<input type="text" name="age"><br>
    邮箱：<input type="text" name="email"><br>
    <input type="submit">
</form>

@RequestMapping("/testpojo")
public String testpojo(User user){
     System.out.println(user);
     return "success";
}

7、解决获取请求参数的乱码问题

解决获取请求参数的乱码问题，可以使用SpringMVC提供的编码过滤器CharacterEncodingFilter，但是必须在web.xml中进行注册

加载时间：监听器(最早)->过滤器->servlet，因为监听器在监听完ServletContext创建和销毁只执行一次，而过滤器在一直执行，每个请求都要经过过滤器，因此使用过滤器来修改编码

<!--配置springMVC的编码过滤器-->
<filter>
    <filter-name>CharacterEncodingFilter</filter-name>
    <filter-class>org.springframework.web.filter.CharacterEncodingFilter</filter-class>
    <init-param>
        <param-name>encoding</param-name>
        <param-value>UTF-8</param-value>
    </init-param>
    <init-param>
        <param-name>forceResponseEncoding</param-name>
        <param-value>true</param-value>
    </init-param>
</filter>
<filter-mapping>
    <filter-name>CharacterEncodingFilter</filter-name>
    <url-pattern>/*</url-pattern>
</filter-mapping>
