3.配置web.xml

主要负责注册servlet,过滤器，监听器等等

  • 默认配置

    使用此配置作用下，SpringMVC的配置文件就会默认位于WEB-INF下，默认名称为下面的””<servlet-name>中的值””+”-servlet.xml”，例如，以下配置所对应SpringMVC的配置文件位于WEB-INF下，文件名为springMVC-servlet.xml,但这种方式并不好，SpringMVC的配置文件，应放在resource资源文件夹下，WEB-INF应该只存放页面文件例如***.html等，因此便产生了拓展配置，可以改变SpringMVC配置文件的位置

    <?xml version="1.0" encoding="UTF-8"?>
<web-app xmlns="http://xmlns.jcp.org/xml/ns/javaee"
         xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
         xsi:schemaLocation="http://xmlns.jcp.org/xml/ns/javaee http://xmlns.jcp.org/xml/ns/javaee/web-app_4_0.xsd"
         version="4.0">
    <!--配置SpringMVC前端控制器，对前端的请求做统一管理-->
    <servlet>
        <servlet-name>SpringMVC</servlet-name>
        <!--选择SpringMVC自带的前端控制器DispatcherServlet-->
        <servlet-class>org.springframework.web.servlet.DispatcherServlet</servlet-class>
    </servlet>
    <servlet-mapping>
        <servlet-name>SpringMVC</servlet-name>
        <!--因为是对所有请求进行处理，"/"代表当前浏览器所发生的所有请求
        例如/login或.js,.css等等的请求路径,但不包括对.jsp文件的请求路径-->
        <url-pattern>/</url-pattern>
    </servlet-mapping>
</web-app>

  • 拓展配置

    可通过init-param标签设置SpringMVC配置文件的位置和名称，通过load-on-startup标签设置SpringMVC前端控制器DispatcherServlet的初始化时间

    <?xml version="1.0" encoding="UTF-8"?>
<web-app xmlns="http://xmlns.jcp.org/xml/ns/javaee"
         xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
         xsi:schemaLocation="http://xmlns.jcp.org/xml/ns/javaee http://xmlns.jcp.org/xml/ns/javaee/web-app_4_0.xsd"
         version="4.0">
    <!--配置SpringMVC前端控制器，对前端的请求做统一管理-->
    <servlet>
        <servlet-name>SpringMVC</servlet-name>
        <!--选择SpringMVC自带的前端控制器DispatcherServlet-->
        <servlet-class>org.springframework.web.servlet.DispatcherServlet</servlet-class>
        <!--使用init-param标签(初始化参数)来改变SpringMVC配置文件(即spring的配置文件)的存放路径和名称
        参数：
        <param-name>:contextConfigLocation(上下文路径)->固定为这个
        <param-value>:需使用classpath来代表在main路径下的文件，填写好对应的名称(与自己创建配置文件时起的名称一致便可)
        -->
        <init-param>
            <param-name>contextConfigLocation</param-name>
            <param-value>classpath:SpringMVC.xml</param-value>
        </init-param>
        <!--SpringMVC的配置文件初始化默认是在浏览器的第一次请求时，如果有大量的配置文件需要初始化则会严重影响浏览器第一次时的速度
        因此使用<load-on-startup>标签将DispatcherServlet的初始化时间提前到服务器启动时
        -->
        <load-on-startup>1</load-on-startup>
    </servlet>
    <servlet-mapping>
        <servlet-name>SpringMVC</servlet-name>
        <!--因为是对所有请求进行处理，"/"代表当前浏览器所发生的所有请求
        例如/login或.js,.css等等的请求路径,但不包括对.jsp文件的请求路径-->
        <url-pattern>/</url-pattern>
    </servlet-mapping>
</web-app>

4.创建请求控制器

创建类com.hong.controller.***.java

使用注解@Controller和在SpringMVC.xml中配置扫描的方式将其注册到spring中交由spring管理，并在SpringMVC.xml中配置好视图解析器

HelloController->

//@Controller表示交由Spring管理
@Controller
public class HelloController {
    
}

SpringMVC.xml->

<?xml version="1.0" encoding="UTF-8"?>
<beans xmlns="http://www.springframework.org/schema/beans"
       xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
       xmlns:context="http://www.springframework.org/schema/context"
       xsi:schemaLocation="http://www.springframework.org/schema/beans http://www.springframework.org/schema/beans/spring-beans.xsd http://www.springframework.org/schema/context https://www.springframework.org/schema/context/spring-context.xsd">

        <!--扫描指定路径下的类，并对拥有对应注解的类进行注册管理-->
       <context:component-scan base-package="com.hong.controller"></context:component-scan>

        <!--视图解析器配置，负责最终的页面跳转,即配置后在使用页面跳转时，只需填写页面名称便可，即想跳转indx页面,return "index"便可，无需填写具体路径及后缀,由视图解析器自动帮你配置-->
        <!-- 配置Thymeleaf视图解析器
        class->选择ThymeleafViewResolver （Thymeleaf视图解析器）
        参数:
        order->配置视图解析器的优先级(因为可以设置优先级代表着可以使用多个视图解析器为其分配不同的优先级来进行使用)
        characterEncoding->编码
        templateEngine->模板(选择对应模板并进行配置)->
        -->
        <bean id="viewResolver" class="org.thymeleaf.spring5.view.ThymeleafViewResolver">
            <property name="order" value="1"/>
            <property name="characterEncoding" value="UTF-8"/>
            <property name="templateEngine">
                <bean class="org.thymeleaf.spring5.SpringTemplateEngine">
                    <property name="templateResolver">
                        <!--
                            {(内部bean为templateEngine赋值)
                            classs:SpringResourceTemplateResolver
                            perfix：视图前缀
                            suffix:视图后缀
                            (这里前后缀即xxx文件夹下的xxx.html文件)
                            templateMode：模板类型
                            characterEncoding:编码
                            }
                        -->
                        <bean class="org.thymeleaf.spring5.templateresolver.SpringResourceTemplateResolver">
                            <!-- 视图前缀 -->
                            <property name="prefix" value="/WEB-INF/templates/"/>
                            <!-- 视图后缀 -->
                            <property name="suffix" value=".html"/>
                            <property name="templateMode" value="HTML5"/>
                            <property name="characterEncoding" value="UTF-8" />
                        </bean>
                    </property>
                </bean>
            </property>
        </bean>
</beans>

5.1实现对首页的访问

在HelloWord.java中写好测试代码

@Controller
public class HelloController {

    //@RequestMapping 请求映射 作用为请求与对应的请求方法创建映射关系
    //@RequestMapping注解的value属性可以通过请求地址匹配请求，/表示的当前工程的上下文路径
    //即请求localhost:8080/springMVC/便可访问该请求方法
    @RequestMapping("/")//如果只为value属性赋值时value可以不写
    public String index(){
        //如果没有配置视图解析器，则需return "/WEB-INF/templates/index.html"
        return "index";
    }

@Controller
public class HelloController {

    //@RequestMapping 请求映射 作用为请求与对应的请求方法创建映射关系
    //@RequestMapping注解的value属性可以通过请求地址匹配请求，/表示的当前工程的上下文路径
    //即请求localhost:8080/springMVC/便可访问该请求方法
    @RequestMapping("/")//如果只为value属性赋值时value可以不写
    public String index(){
        //如果没有配置视图解析器，则需return "/WEB-INF/templates/index.html"
        return "index";
    }

    @RequestMapping("/target")//请求路径
    public String toTarget(){
        return "target";
    }
}

6.总结

浏览器发送请求，若请求地址符合前端控制器的url-pattern，该请求就会被前端控制器DispatcherServlet处理。前端控制器会读取SpringMVC的核心配置文件，通过扫描组件找到控制器，将请求地址和控制器中@RequestMapping注解的value属性值进行匹配，若匹配成功，该注解所标识的控制器方法就是处理请求的方法。处理请求的方法需要返回一个字符串类型的视图名称，该视图名称会被视图解析器解析，加上前缀和后缀组成视图的路径，通过Thymeleaf对视图进行渲染，最终转发到视图所对应页面

即：

1.向服务器请求(请求地址)

2.前端控制器获得请求与自身配置(web.xml中)中的url-patten进行匹配，符合则由前端控制器处理

3.前端控制器通过扫描包<context:component-scan base-package=”xxx”>找到控制器(@Controller)

4.与在与控制器中@RequestMapping的value值相匹配

5.匹配成功，由该方法返回字符串类型的视图名称(return “index”;)，该视图名称会被视图解析器解析，加上前缀和后缀组成视图的路径，通过Thymeleaf对视图进行渲染，最终转发到视图所对应页面
